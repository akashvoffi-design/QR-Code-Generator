## ✨ Features

| Category | Details |
|---|---|
| 🎨 **Glassmorphism UI** | Frosted glass aesthetic with backdrop blur, gradients, and smooth animations |
| ⚡ **Instant Generation** | Real-time QR creation from any URL or text input |
| 💾 **Download as PNG** | One-click export of your QR code image |
| 📋 **Copy to Clipboard** | Paste the QR image anywhere without saving |
| 🔁 **Keyboard Support** | Press `Enter` to generate — no mouse needed |
| 📱 **Fully Responsive** | Looks great on desktop, tablet, and mobile |
| 🛡️ **Error Correction** | High-quality output using error correction level `H` |
| 🚫 **Zero Dependencies** | Single HTML file — no npm, no build step |

## 🚀 Getting Started

### Clone & Open

```bash
git clone https://github.com/akashvoffi-design/QR-Code-Generator.git
cd QR-Code-Generator
```

Then simply open the file:

- 🖱️ **Double-click** `qr.html` to open in your default browser
- Or right-click → **Open with** → your preferred browser

That's it. You're live. ✅

---

## 🧭 How to Use

```
1. 📝  Enter a URL or any text in the input field
2. ⚡  Click "Generate" or press Enter
3. 👀  Preview your QR code instantly
4. 💾  Hit "Download" to save as PNG
5. 📋  Or hit "Copy" to copy directly to clipboard
6. 🔁  Click "Clear" to start fresh
```

## 💡 Under the Hood

<details>
<summary><b>🔲 QR Code Generation</b></summary>

Uses **QRCode.js** to render high-quality QR codes with error correction level `H` — meaning up to 30% of the code can be damaged and still scan correctly.

</details>

<details>
<summary><b>🎨 Glassmorphism UI</b></summary>

Built using pure CSS with `backdrop-filter: blur()`, semi-transparent backgrounds, and layered gradients — no UI framework needed.

</details>

<details>
<summary><b>📋 Copy to Clipboard</b></summary>

Uses the native **Clipboard API** to write the QR image directly — no temporary downloads or canvas hacks required.

</details>

<details>
<summary><b>📱 Responsive Design</b></summary>

Built with **Flexbox** and viewport-relative units for seamless adaptation across all screen sizes.

</details>
